# README Section Suggestions

## Recommended order

1. Project title and one-line purpose
2. What this is / What this is not
3. Quick start
4. Core usage pattern
5. Skill catalog or feature list
6. Interoperability and compatibility notes
7. Contribution notes (optional)

## Writing rules

- Keep sections short and concrete.
- Prefer command snippets over long prose.
- State assumptions and non-goals explicitly.
- Avoid framework-like language unless strictly required.
